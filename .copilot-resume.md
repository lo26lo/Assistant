## 🔴 PIÈGES CRITIQUES (lire en premier)

1. **vcvarsall.bat** → `C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools\...` PAS `Community`
2. **NE PAS supprimer** `build\vcpkg_installed\` — onnxruntime CUDA+TRT a pris ~2h à compiler
3. **NE PAS utiliser** `spdlog::flush_every(seconds(0))` → crash ACCESS_VIOLATION (spin-loop)
4. **NE PAS appeler** `setupLogging()` dans Application — déjà fait dans `main.cpp`
5. **NE PAS utiliser** Ninja comme générateur CMake → bug `rules.ninja` CMake 4.2.3
6. **Atomic** : utiliser `m_frameCount.exchange(0)` PAS `m_frameCount = 0`
7. **IBomData.h** : `BoardInfo` a `boardBBox` (BBox struct minX/minY/maxX/maxY), PAS `width`/`height`
8. **`parseCommandLine()`** : désactivé — `QCommandLineParser::process()` sur app Windows GUI cause QMessageBox+exit
9. **QApplication** : créé dans `main.cpp` AVANT `Application` (QObject). Application reçoit `QApplication&`
10. **iBOM JSON coords** : TOUJOURS des arrays `[x, y]`, PAS des objets `{x:..., y:...}`. Utiliser `readPoint()` helper.
11. **iBOM value/footprint** : les objets `footprint` ne contiennent PAS `val`/`footprint` → ces infos sont dans `bom.fields`. Cross-référence dans `parseString()` après `parseBomGroups()`.
12. **Git** : `.gitignore` configuré — `build/`, logs, IDE, models ONNX exclus. Remote: `lo26lo/Assistant` branche `main`.
13. **Build corruption** : Si `nmake` est interrompu pendant le link, le `.exe` est corrompu (« endommagé et illisible »). Fix : supprimer le `.exe` + relancer `nmake`. Si MOC obj vide : supprimer `MicroscopeIBOM_autogen/timestamp` + le `.obj` vide.
14. **Tuer l'app avant rebuild** : Le linker échoue LNK1104 si `MicroscopeIBOM.exe` est encore en cours d'exécution. Toujours `Stop-Process` avant `nmake`.
