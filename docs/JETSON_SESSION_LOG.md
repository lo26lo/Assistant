3. **Inspection Progress (suite 130 / ERREUR #40)** : charger un iBOM → le panneau affiche « 0/N (0%) » (plus « No inspection data ») ; cliquer Placed → compteur avance ; relancer l'app → progression restaurée.
4. **Letterbox (suite 129)** : dès qu'un modèle `.onnx` existe, vérifier au log/overlay que les détections tombent bien sur les composants (les coordonnées passent par le nouveau mapping ; le chemin blob n'est pas concerné).
5. **GitHub Actions** : activer Actions sur le repo (Settings → Actions) — le job `unit-tests` (suite 129, validé en local) doit passer au vert sur le prochain push.
6. **Face arrière (suite 131)** : build (Application/ControlPanel/BomPanel touchés, non compilables hors Jetson) puis scénario : charger un iBOM double-face → cocher « Back side (board flipped) » → l'alignement se reset + statut « Back side active » ; retourner physiquement la carte → **Auto-Align** doit verrouiller sur la face arrière (blobs ou modèle) avec l'overlay **miroité, labels lisibles** ; clic sur un composant → sélectionne le bon composant back ; le filtre BomPanel suit ; 4-coins/2-comp/anchor marchent aussi côté back (coins cliqués « tels que vus ») ; recocher Front → re-align avant, la restauration d'alignement sauvegardé ne se propose que côté Front.
7. **Hot-path GUI zéro-copie (suite 132)** : (a) l'image caméra s'affiche avec les **bonnes couleurs** (BGR888 natif — une inversion rouge/bleu = régression immédiate à signaler) sur les 3 chemins : direct, calibré/undistort, RealSense ; (b) le flux RemoteView garde les bonnes couleurs (encodage JPEG) ; (c) **mesurer** : CPU du thread GUI en baisse vs avant (~5-6 ms/frame de cvtColor+memcpy supprimés à 1080p) ; (d) ⚠️ point de vigilance QOpenGLWidget : si le moteur GL re-convertit le BGR888 à chaque upload de texture (coût réintroduit DANS Qt), plan B = revenir à RGB888 mais convertir dans le **thread capture** — à trancher à la mesure.
8. **Auto-Align carte nue (suites 136-137 / ERREURS #57-#58)** : rebuild puis, carte **nue** sous la D405, Reset Alignment → **Auto-Align**, carte dans une orientation **quelconque** (pas seulement nominale — le seeding #58 doit gérer 90/180°) : attendu = lock `re-anchored on N/M pads` à **ratio ≥ ~0.6** et overlay au bon endroit, **ou** refus honnête (`ambiguous registration … refusing to guess` / `inlier ratio too low`) → alignement manuel 2 points. **Anti-« clack »** : après un alignement parfait (manuel ou auto), laisser tourner le re-anchor périodique ≥ 1 min — l'overlay ne doit **jamais** sauter ; un alias tenté loggue `correction exceeds healthy-tracking cap`. Soulever/reposer la carte → récupération Lost normale (le cap ne s'applique pas à Lost). Vérifier le **temps** d'un Auto-Align (2 bootstraps ×2 phases — si > ~4 s sur Orin, le noter). Carte **peuplée** : le chemin composants doit encore gagner. Molettes si trop strict : `minInlierRatio` 0.4, `bootstrapAmbiguityRatio` 0.97, gate 5 mm, cap 12 mm — logguer le message exact avant de retoucher. **Suite 138** : à chaque essai, récupérer `$IBOM_DATA_DIR/debug/reanchor_*.jpg` (détections MSER rouges / détections **pad magenta** / pads projetés verts) — c'est la source de vérité pour diagnostiquer, pas la capture d'écran ; et **éclairer la carte** (le 2e essai terrain était quasi sous-exposé → blobs de bruit). **Suite 140** : vérifier au log `[pad-detect] N pad-like blobs` — N doit approcher le nombre de pads visibles ; les cercles magenta du dump doivent tomber sur les pads réels. Si oui mais que l'alignement aliase encore → c'est la symétrie du layout, décision modèle (P0).
9. **Minimap V2 (suite 145)** : charger un iBOM → zoom molette (ancré au curseur), pan au drag gauche une fois zoomé, ⌂ = fit ; touche **M** (ou ⛶, ou double-clic) ouvre la grande vue redimensionnable (Esc ferme, interactions identiques dedans) ; **drag du rectangle FOV** → l'overlay se re-ancre au relâché ; **Ctrl+clic** sélectionne le composant (la ligne BOM défile jusqu'à lui) ; ◎ suit la caméra (inset global en bas-droite) ; la **couverture verte** s'accumule en balayant la carte (toggle/reset au menu contextuel droit) ; FOV **orange** quand le tracking faiblit (<15 inliers ou reproj >6 px), **rouge** quand Lost ; cocher « Back side » → la carte montre les composants **back** (fix de la sync layer manquante) ; perfs : aucune saccade GUI à 30 fps sur carte dense (cache pixmap — vérifier surtout en mode follow) ; dock élargissable au-delà de 360 px et détachable (menu contextuel).

## État actuel — au 2026-07-07 (FACE ARRIÈRE bout en bout — le plus gros trou produit comblé)

> - **`ComponentReanchor`** : `estimate()`/`bootstrap()` travaillent en « repère vue » (x négé quand `activeLayer == Back`, l'axe du miroir est indifférent — la translation du fit absorbe l'offset) et **recomposent le miroir** dans la matrice retournée (`H_raw = H_vue · M`).
